## Build Commands

```bash
# Build (release)
cargo build --release

# Build (debug)
cargo build

# Run tests (most tests are ignored by default as they require server/keychain)
cargo test

# Run a specific test
cargo test test_name

# Run ignored tests (requires running server and keychain access)
cargo test -- --ignored
```

## Architecture

VT (Vault) is a macOS-based KMS (Key Management System) that uses the system keychain for secure secret storage and encryption.

### Components

- **main.rs**: CLI entry point using clap. Defines subcommands: `serve`, `init`, `create`, `read`, `inject`, and `secret` (export/import/rotate-passcode). macOS-specific commands are conditionally compiled.

- **serve.rs**: Axum HTTP server providing `/encrypt` and `/decrypt` endpoints. Uses middleware for auth (request/response body encryption with VT_AUTH token) and logging. Requires local biometric authentication for decrypt operations.

- **cli.rs**: Client-side logic. `VTClient` makes authenticated requests to the server with encrypted bodies. Handles secret creation, reading, and injection into env vars/files/commands.

- **security.rs**: Cryptographic operations using AES-256-GCM. Manages keychain storage (`passcode` and `passphrase` secrets), key derivation, and macOS local authentication via Touch ID/password.

### Secret Management Flow

1. `vt init` creates two keychain entries:
   - `passcode`: Random bytes + auth_token, used to derive the passphrase encryption key
   - `passphrase`: The actual encryption key (encrypted with derived key from passcode + USER + binary path)

2. The server loads these on startup and uses them for all crypto operations

3. Clients use `VT_AUTH` env var to encrypt request/response bodies when communicating with the server

### VT Protocol Format

`vt://{location}/{type}{data}` where:
- `location`: Currently only "mac" supported
- `type`: "0" for raw secrets, "1" for TOTP
- `data`: Base64 URL-safe encoded encrypted data

### Key Environment Variables

- `VT_ADDR`: Server address (default: 127.0.0.1:5757)
- `VT_AUTH`: Authentication token generated by `vt init`
- `RUST_LOG`: Log level (debug in dev builds, info in release)
